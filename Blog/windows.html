1. Windows启动问题

1.1 系统蓝屏：Unmountable_boot_volume

笔者用迅雷下载一个大文件的时候，由于下载速度过快导致系统一直处于假死机的状态，于是手贱地强制关机，于是悲剧发生了：
当我重启计算机的时候，系统蓝屏，提示Unmountable_boot_volume。并且无论如何也修复不了。

网上搜索解决办法，分别尝试了以下几种方法：

(1)进入BIOS确认是否识别出硬盘，若识别出硬盘则说明硬盘没坏，拆机重新插拔硬盘断电。


	不成功。进入BIOS后发现硬盘并没有损坏，但是给硬盘断电后仍然蓝屏。问题不在硬盘上，而在系统上。

(2)进入安全模式，修复系统。


	不成功。根本无法进入安全模式。看来问题比较严重。

(3)chkdsk。

step-1: Start your computer using a Windows 10 DVD or a USB drive. If you don't have any Windows 10 media available, visit this page and follow the steps to create one using the Media Creation Tool.

step-2: Once starting using a DVD or USB device, select the "Repair your computer" option located at the lower-left corner.

step-3: Choose "Command prompt", type chkdsk /r C: then press "Enter". If your bootable drive is a letter other than "c", replace "C:" with the bootable drive you use.

step-4: Be patient while chkdsk checks your drive for errors. It could table a really long time.


	不成功。检查到错误，一些位置无法读取，修复失败。重启仍然蓝屏。

(4)Automatic Repair。

step-1: Start your computer using a Windows 10 DVD or a USB drive. If you don't have any Windows 10 media available, visit this page and follow the steps to create one using the Media Creation Tool.

step-2: Once starting using a DVD or USB device, select the "Repair your computer" option located at the lower-left corner.

step-3: Choose "Troubleshot" > "Advanced options" > "Automatic Repair".

step-4: Select the OS you wish to repair. Be patient while the utility attempts to repair the files.


	不成功。自动修复失败。

无奈之下，只能采取重装系统的下策了。

(5)最终方案：重装系统。

step-1: 在 微软的官方网站下载并制作了启动盘。

step-2: 打开BIOS设置，选择与硬盘分区表格式一致的引导方式，启动U盘中的系统。

step-3: 建议先尝试修复，若修复未遂，再重装系统。

step-4: 重装时最好选择升级，若不支持升级，就把系统装在C分区覆盖原来的系统。之后C区会被清理，但是其他分区的数据得以保存。

安装完成后，拔掉U盘重新启动计算机，然后设置新的系统。

这是下策，但是也是终极解决方案。

关于step-2的说明：win8/win10系统均添加快速启动功能，预装的win8/win10电脑默认都是UEFI引导的GPT硬盘。 传统的引导方式为Legacy引导的MBR硬盘。UEFI必须跟GPT对应，同理Legacy必须和MBR对应。 如果引导方式和分区格式不对应，则会遇到"无法在驱动器0的分区1上安装"的问题。
